Part 1: Cookie Hit Counter

Original Description: Create a web app that keeps track of user's hits to the site, their IP, and current time zone.

Interpretation: This part was quite straightforward and implemented the same way.

Implementation: I made use of ASP.NET's cookies API and they're API for obtaining IP Addresses. The local time was done on the javascript side.

Guide: Click refresh to update the counter. Click Reset to set the counter back to 0

Part 2: Slide Show

Interpretation: This part was easier to implement than the assigment 2 version! The instructions were straightforward and implented as instructed

Part 2!

Part 3: Computer Store (First Implementation)

Original Description: Create a web app is similar to a computer store, where a user login, create, configure, and buy both computer parts and entire computers.

Interpretation: This version of the store, is hardcoded and does not make use of a database. This is quite difficult with the amount of data being processed and so using cookies was a challenge. In this verion you can add products and computers to your cart, login/create users, and configure a laptop.

Implementation: I made use of many cookies and helper classes (Constants.css) to try and manage the data. All images where taken from iconsfinder.com.

Guide: Navigate to Products > Computers to see the list of laptops available. You can click and configure them before adding them to your cart. Products > Computer Parts shows you the list of parts which you can add to your cart by clicking on the image button. The contact tab shows a list of contacts. The feedback form will validate your input before submission. The login/create user can be accessed by clicking either the "LOG IN/SIGN UP" or your username. This page will validate your login/ sign up and allow you to log out.

Part 3!

Part 4: Final Computer Store

Original Description: Create a web app that implements database driven computer store (upgrade from Part 3)

Interpretation: This was much easier to implement than part 3. The store can recover your password, and smartly display the cart items. It also keeps track of everything using a database This reduces the need for cookies and extra data.

Implementation: My database was created as seen in tableSetup.sql. This allowed me to easily access the products, orders, and customers. Most of the same implementation was used other than it coming from a database. That being said, I had to dynamically create controls in the C# serer code based on the queried items. This was a bit difficult but after doing it a few times, it became easier to handle.

Guide: Similar to Part 3. New features are the ability to recover your password (accessed on the log in page). THe cart is much nicer and has the ability to update the qty of the items and will update your cart/ database accordingly.
