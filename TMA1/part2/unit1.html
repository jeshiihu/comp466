The Web

Introduction

When you surf the internet what types of things do you think occurs in the background? How does it know what to display? How does it know when you click or input items? All of these 'things' are a result of a web system in which a web server and web client communicate with each other.

  • Web Server: Processes client (user) input and responds to their requests
    • returning documents and files to client
    • retreiving and committing data to client (REST)
  • Web Client: sends requests to the server and is usally known as the 'web browser'

This can all be done through HTTP

  • HTTP: HyperText Transfer Program, transfers files and requests over internet/intranet

HTTP

All websites you have come across essentially are made up of HTML files and are accessed using URIs/URLs

  • HTML: HyperText Markup Language, this describes the content of what to be dislplayed to a web page. These files have the extention .html or .htm
  • URI: Uniform Resource Identifiers, in general they identify resources on the internet
  • URL: Uniform Resource Lookup, these are URIs that begin with http:// and commonly involve file, directory, internet searches and/or database lookups

In order for a web page to display commication must be performed between the client and the server. First the client (user) would type in a URL into a web browser with HTTP. This results in a request being sent to the webpage of the address. The server would then receive this request and lookup the resource that the client has requested for. After finding the resource the server will respond to the client with the resources' contents. The following images (Deital p.607-608) explains this process quite nicely.

There are various request types such as GET and POST. GET asks the server to retrieve the the information such as data, images, or files. POST usually will send data to the server where it can either add it to a database or process the information.

Typically in a web system, you'll find a multi-tier architecture. This will be comprised of a bottom tier (database layer), middle tier (business controller logic), and lastly the top tier/ client tier (GUI layer).

HTML5 Basics

After an introduction to what goes on behind the scenes of a webpage, you might be wondering about the HTML that make this all possible. If you have experience with programming languages such as C, C++, C#, and Java, you might think HTML should be a similar enough. Wrong! HTML5 is a markup language that specifies both the structure as well as the content of a web page.

The most basic structure of an HTML5 file will be as follows (Deital p.39)

This is the skeleton that makes up any and all HTML5 files you view in a web browser today. We will quickly break down what the above file is composed of.

  • Line 1: This is the Document type, web browsers require this line in order for it to know what type of file it will be processing (compatibility etc.)
  • Lines 2-3: These lines are comments and will be ignored by the browser
  • Lines 6-9: This tag element is known as the Head. It contains the information of the HTML5 for the browser to properly know what to do with the file (encodings, title, style CSS, etc.)
  • Lines 11-13: This is the Body, where all of a web page's content is written.

To break down the terminology of the structures in an HTML5 file further, the following list will provide a basic syntax:

  • Element: As seen above, the head and body elements are required in every HTML5. An element is usually in the form <tagname>content</tagname> (paragraphs, list items, headers, etc.). They can also be written without an end tag when no content is required and will be in the form <tagname/>
  • Nested Elements: These are known as elements that fall within another element (ie. the title element at line 8)
  • Attributes:Within the starting tag of an element, you can specify properties of the element using attributes. These follow the form attributeName="attributeValue". Common attributes include the class (grouping), id (unique identifier), and src (for images and other media).

CSS3

Introduction

Now that we know how to create the contents of a webpage, how do we make it look nicer than plain black and white text with no formatting? This is done using CSS, Cascading Style Sheets.

Although you can add formatting to an HTML5 directly with attributes, this can result in messy code. It also makes it difficult to make any uniform change to the same element for example (unnecessary duplicaate formatting). For example, you want all headers to be blue but done want to add that color attribute to every single element in all of your files. CSS takes care of all formatting in a seperate file where you can easily control all of the same elements' styling.

In order to link a style sheet to your HTML5 file, you need to add the following line within the head element. <link rel="stylesheet" type="text/css" href="filePath.css">. Now you may begin creating your .css file that will help format your web page

General Guidelines

Similar to how there is so many attributes that you can do, it would be impossible for us to cover everything. However there is a general format that can be followed for the CSS styling. First we typically start with the tagname, class name, or id name in which we wish to stylize.

  • Tags can simply be written as they are (ie. h2 {})
  • Classes need to being with a '.' (ie. .myClass {})
  • IDs need to begin with '#' (ie. #uniqueTitle {})

You can also add multiple elements by comma seperating them (ie. h1, h2, #firstTitle {}) as well as follow a nested element such as a paragraph within the class "section"
.section p {}

Within the curly brackets is where you will add the attributes and their values. These follow the form attributeName: attributeValue;

Common Attributes

The following are a few common attributes that you can do with CSS

  • Color: You can set the value of the color attribute as a common color or an rgb hex value. The common colors can be written in plain english (ie. grey, red, green) and a list can be found here. If you wish to use your own colours you may use the hex value with # appended in the beginning (ie. color: #00BFFF;)
  • Width/Height: The size of an element can be modified. You can use various ways such as specifying the exact pixel (ie. 50px), relative to the font size (ie. 3em), percentage (ie. 50%), etc.
  • Margins/Padding: You can change the margins of the element or the padding in a similar way as the width and height
  • Alignment: You can center, left, right, top, or bottom align. It is easiest in my opinion to text-align the containing element if you wish to, for example, center align the nested element.
  • Border: You can change a border's size and colour or even get rid of one (setting it to 0)
  • Display: There are a few display attribute values you can use such as inline, inline-block, block, list-item, etc.

The list goes on and on. Again, the best way in my opinion is to research what you want to do and how it can be doen with CSS!
